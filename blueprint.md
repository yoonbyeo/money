# TYPELAB: 종합 유형 검사 플랫폼 프로젝트 청사진

## 1. 프로젝트 개요 및 목표

**목표:** 기존의 정적 MBTI 검사 사이트를 전문가 수준의 디자인과 사용자 경험을 갖춘 매력적인 종합 유형 검사 플랫폼으로 전면 개편합니다. 사용자의 만족도를 극대화하고, Google AdSense 승인 기준을 충족시키는 것을 최종 목표로 합니다.

**핵심 전략:**
- **모던 웹 기술 도입:** Web Components, ES Modules, Modern CSS 등 표준 기술을 활용하여 프레임워크 없이도 유연하고 확장 가능한 구조를 구축합니다.
- **사용자 경험(UX) 혁신:** 정적인 설문 형식을 탈피하여, 카드 스와이프, 드래그 앤 드롭 등 동적인 인터랙션을 도입하여 검사 과정의 몰입도를 높입니다.
- **시각적 디자인(UI) 강화:** 기존의 깔끔한 스타일을 기반으로, 고급스러운 애니메이션, 인터랙티브한 차트 및 시각 요소를 더해 전문가 수준의 세련된 디자인을 완성합니다.
- **콘텐츠 및 정책 강화:** AdSense 가이드라인에 부합하도록 개인정보 처리방침, 이용약관 페이지를 별도로 구성하고, 콘텐츠의 독창성과 전문성을 높입니다.
- **Firebase 통합:** 사용자 결과 저장, 계정 시스템 등 미래의 확장성을 고려하여 Firebase와의 연동을 준비합니다.

## 2. 디자인 및 기능 명세

### 2.1. 스타일 및 디자인 원칙
- **Color Palette:** 기존의 따뜻한 배경색(`--bg: #f6f3ee`)과 악센트 컬러(`--accent: #ef6f3b`)를 유지하되, 깊이감과 생동감을 더할 수 있는 보조 색상을 추가합니다.
- **Typography:** 가독성이 높은 'Manrope'와 클래식한 'Playfair Display' 폰트 조합을 그대로 사용하여 안정감을 줍니다.
- **Layout:** 모바일과 데스크톱 모두에서 최적화된 반응형 레이아웃을 유지하며, 카드형 UI와 그리드 시스템을 적극 활용합니다.
- **Visual Effects:**
  - **그림자(Shadow):** 컴포넌트의 깊이감을 표현하기 위해 부드럽고 깊은 다층 그림자 효과(`box-shadow`)를 적용합니다.
  - **배경(Background):** 미묘한 노이즈 텍스처와 은은하게 움직이는 그라데이션 오브(`orb`)를 사용하여 고급스러운 느낌을 더합니다.
  - **애니메이션:** 페이지 전환, 스크롤, 버튼 호버 등 사용자 인터랙션에 부드러운 애니메이션을 적용하여 생동감을 부여합니다.

## 3. 개발 계획 (로드맵)

### **Phase 1: 프로젝트 구조 개편 및 기반 설정 (완료)**
1.  **`blueprint.md` 파일 생성:** 프로젝트의 목표와 계획을 정의. (완료)
2.  **파일 구조 재구성:**
    - `main.js`를 `app.js`(메인 로직), `data/questions.js`, `data/answers.js`, `data/results.js`로 분리. (완료)
    - `components` 디렉토리 생성 및 Web Component 파일들(`QuestionCard.js` 등) 추가. (완료)
3.  **Firebase 통합 설정:**
    - `.idx/dev.nix` 파일에 `pkgs.nodejs` 추가 (이미 존재했음). (완료)
    - `.idx/mcp.json` 파일에 Firebase 서버 구성을 추가하여 개발 환경 준비. (이미 존재했음). (완료)
4.  **기본 HTML/CSS 구조 업데이트:** 신규 파일 및 컴포넌트 구조에 맞게 `index.html`과 `style.css`를 수정. (완료)

### **Phase 2: 핵심 기능 재설계 및 구현 (완료)**
1.  **Web Component 개발:**
    - `<question-card>`: 스와이프 가능한 질문 카드 컴포넌트 개발. (완료)
    - `<result-chart>`: Chart.js 또는 유사 라이브러리를 활용한 결과 차트 컴포넌트 개발. (완료)
    - `<type-badge>`: MBTI 유형을 표시하는 뱃지 컴포넌트 개발. (완료)
2.  **인터랙티브 검사 로직 구현:**
    - `app.js`에서 카드 스와이프 이벤트를 감지하여 사용자 답변을 저장하는 로직 구현. (완료)
    - 검사 진행 상태를 시각적으로 보여주는 프로그레스 바 업데이트. (완료)
3.  **결과 페이지 동적 렌더링:**
    - 계산된 점수를 바탕으로 `<result-chart>`와 `<type-badge>`를 포함한 결과 페이지를 동적으로 생성. (완료)

### **Phase 3: 콘텐츠 및 정책 강화 (완료)**
1.  **정책 페이지 생성:**
    - `privacy.html`, `terms.html`, `policy.html` 파일 생성 및 표준 약관 내용 작성.
    - `index.html`의 푸터 및 관련 섹션에 해당 페이지로의 링크 추가. (완료)
2.  **AdSense 최적화:**
    - 사이트 전반의 콘텐츠 품질을 검토하고, 명확한 네비게이션 구조를 보장.
    - 생성된 정책 페이지가 쉽게 접근 가능한지 확인. (완료)
3.  **최종 테스트 및 배포:**
    - 모든 기능이 정상적으로 동작하는지, 모든 링크가 유효한지 전반적으로 테스트.
    - 사용자 피드백을 받을 준비. (대기)
